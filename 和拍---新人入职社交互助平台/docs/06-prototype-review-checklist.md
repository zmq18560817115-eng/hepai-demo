# 06 · 原型评审清单

> 评审会议前打印或勾选；每项需 **演示可验证**（非仅文档存在）。

## A. 体验与文案（减压导向）

- 盲盒文案无「性格好坏」评判，选项中性
- 隐私承诺在盲盒页可见，说明 30 天面具机制
- 能量偏低提示为建议式（「可以试试…」）而非警告式
- 无「未完成扣 KPI」类表述
- 中英文混排（如 `MATCHING`）是否统一——产品拍板

## B. 流程与步骤数

- 首次入职 ≤ 3 题 + 1 次开盒（总点击 ≤ 5）
- 午餐匹配演示 ≤ 3 步（发起 → 等待 → 成功）
- HR 切换无需重新登录（原型角色切换即可）
- 盲盒未完成时无法跳过到底栏其他页

## C. 数据贯通（核心）

- 盲盒答案 → 生成面具名 / 标签 / 格言
- 安全屋展示与盲盒结果一致
- 刷新页面后面具与能量仍保留（localStorage）
- 导师工作台能看到同一面具标签（mock 新人）

## D. 角色与权限

- 新人 / 导师 / HR 三角色入口清晰
- HR 页无新人底部导航
- 导师页看不到其他导师的私密答题原文
- C 端不能访问 HR 路由（切换才进入）

## E. 载体与钉钉

- 390px 手机框内无横向滚动条
- 主按钮拇指可触达（底部区域）
- 已记录「暂不申请 camera / mic / geo」决策

## F. 与后端契约

- `Persona` 字段与 API 文档一致：`name`, `tags`, `motto`
- 情绪能量对应 `MoodLog.energy_level`
- HR 图表字段名与 `GET /api/hr/`* 设计一致
