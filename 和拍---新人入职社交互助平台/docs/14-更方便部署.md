# 和拍 · 更方便的部署方式

> 按 **操作难度从低到高** 排列。大多数场景 **优先用 Docker 或 Zeabur**。

---

## 方式对比（一张表）

| 方式 | 难度 | 费用 | 固定网址 | 电脑要开 | 适合 |
|------|------|------|----------|----------|------|
| **① Docker 本机** | ⭐ 最低 | 0 | 本机 8080 | 是 | 本机/局域网演示 |
| **② Zeabur 点点部署** | ⭐⭐ | 有免费额度 | ✅ `*.zeabur.app` | 否 | **最省心上云** |
| **③ Render** | ⭐⭐ | 免费档 | ✅ `*.onrender.com` | 否 | 有 GitHub 时 |
| **④ pilot + localtunnel** | ⭐⭐ | 0 | ❌ 会变 | 是 | 临时发链接 |
| **⑤ Oracle 傻瓜版** | ⭐⭐⭐⭐ | 0 | ✅ 自建域名 | 否 | 完全自控 |

## ① Docker 一键（推荐：最简单）

### 准备

安装 [Docker Desktop](https://www.docker.com/products/docker-desktop/)（Mac 点安装，打开一次即可）。

### 一条命令启动

```bash
cd ~/Downloads/text/和拍---新人入职社交互助平台
npm run docker
```

或：

```bash
cd ~/Downloads/text
docker compose up --build -d
```

### 访问

- 地址：http://127.0.0.1:8080/
- 账号：E00001 / 123456

### 常用命令

```bash
cd ~/Downloads/text
docker compose logs -f    # 看日志
docker compose down     # 停止
docker compose up --build -d   # 更新代码后重建
```

### 给别人临时访问

Docker 跑起来后，另开终端：

```bash
npx localtunnel --port 8080 --local-host 127.0.0.1
```

把生成的 `https://xxx.loca.lt` 发出去（链接会变，钉钉需改配置或只聊天发链接）。

## ② Zeabur（推荐：最省心的公网 + 固定域名）

不用自己配 Oracle、不用写 Nginx，**连 GitHub 点几下**。

### 步骤

1. 把 `text` 文件夹推到 **GitHub 私有/公开仓库**（根目录包含 `Dockerfile`、`docker-compose.yml`）。
2. 打开 https://zeabur.com/ 注册（支持 GitHub 登录）。
3. **New Project** → 导入你的 GitHub 仓库。
4. 选择 **Dockerfile 部署**（Zeabur 会自动识别根目录 `Dockerfile`）。
5. 暴露端口 **8080**，等待构建完成。
6. 在 Zeabur 控制台绑定 **免费二级域名**，得到例如：  
   `https://hepai-xxx.zeabur.app`
7. 钉钉开放平台：
   - 首页：`https://hepai-xxx.zeabur.app/`
   - 可信域名：`hepai-xxx.zeabur.app`

### 注意

- 免费额度有限，休眠后首次打开可能慢 10～30 秒。
- SQLite 数据在容器里，重建服务可能重置演示数据（可再访问 `/api/v1/health` 确认后 `db:sync` 逻辑已内置在镜像构建时）。

## ③ Render（适合已有 GitHub）

1. 代码推送到 GitHub（`text` 为仓库根目录）。
2. 打开 https://render.com/ 注册。
3. **New** → **Blueprint** → 连接仓库，Render 会读取 `render.yaml`。
4. 部署完成后得到 `https://hepai-xxxx.onrender.com`。
5. 钉钉填该 HTTPS 地址（与 Zeabur 相同）。

免费实例 **15 分钟无访问会休眠**，首次打开需等待唤醒。

## ④ 本机试点 + 隧道（你之前用过）

```bash
# 终端 A
npm run pilot:dingtalk

# 终端 B（关 VPN 更稳）
npx localtunnel --port 8080 --local-host 127.0.0.1
```

优点：不用 Docker、不用云。缺点：链接常变、电脑要开。

## 怎么选（30 秒决策）

```
只想本机快速跑起来？
  → ① npm run docker

想最少折腾、要固定 HTTPS 给钉钉？
  → ② Zeabur（推 GitHub + 点部署）

已有 GitHub、习惯 Render？
  → ③ Render Blueprint

只演示几小时、零配置？
  → ④ pilot + localtunnel

要完全免费自控、长期稳定？
  → ⑤ Oracle 傻瓜版
```
