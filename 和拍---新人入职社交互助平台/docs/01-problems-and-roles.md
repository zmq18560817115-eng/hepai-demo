# 01 · 问题与角色（1 页）

> 和拍 · 新人入职社交互助平台 · 产品定义摘要

## 我们要解决什么


| 痛点     | 表现                 | 和拍的应对                             |
| ------ | ------------------ | --------------------------------- |
| 入职社交焦虑 | 不敢主动搭话、怕被评判真实性格    | **人格面具（盲盒）**：前 30 天以虚拟化名对外展示      |
| 带教断层   | 不知道找谁、怕打扰 busy 的导师 | **导师库 + 状态灯**：可见「可打扰 / 忙碌」，降低心理门槛 |
| 午餐孤独   | 新人找不到饭搭子           | **蹭饭地图 / 拼图匹配**：轻量、游戏化、无压力邀约      |
| HR 盲区  | 难以及时发现情绪低落的新人      | **B 端仪表盘**：批次活跃度、情绪趋势、风险预警        |


## 三类角色 · 各自目标

### 新员工（C 端 · `newcomer`）

- **想完成**：快速有「被看见」的感觉，找到 1～2 个安全社交入口，不暴露真实性格也能参与互动。
- **成功标准**：完成盲盒 → 进入安全屋 → 至少尝试 1 次情绪记录或午餐匹配。
- **不负责**：带教排班、全公司数据看板。

### 带教导师（C 端 · `mentor`）

- **想完成**：知道哪些新人需要关注、何时可被联系，用最小成本完成带教触达。
- **成功标准**：看到分配给自己的新人列表、面具标签（非私密原文）、情绪异常提示。
- **不负责**：修改 HR 配置、全批次统计。

### HR / 管理员（B 端 · `hr`）

- **想完成**：掌握各批次融入进度与情绪风险，干预有据可依。
- **成功标准**：仪表盘可查看活跃度、情绪曲线、高风险新人清单。
- **不负责**：代替新人完成社交匹配（仅观察与配置）。

## MVP 范围（原型阶段）


| 包含                  | 暂不包含             |
| ------------------- | ---------------- |
| 入职盲盒 → 面具生成 → 安全屋展示 | 真实钉钉 OAuth、IM 打通 |
| 情绪能量条 + 闪光记录入口      | 完整日历 / 皮肤兑换      |
| 午餐匹配演示流             | 真实地理位置匹配算法       |
| HR 静态图表看板           | 导出报表、权限细粒度 RBAC  |
| 导师列表只读 + 占位页        | 导师主动发消息、排班系统     |


## 一句话定位

**专为「社交微恐」职场新人设计的钉钉轻量插件：用游戏化、低暴露成本的互动，把入职前 30 天的社交压力拆成可完成的小步骤。**
