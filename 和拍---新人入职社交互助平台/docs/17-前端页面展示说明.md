# 和拍 · 前端页面展示说明

## 一、本机怎么打开

| 方式 | 命令 | 地址 | 说明 |
|------|------|------|------|
| **演示（推荐）** | `npm run demo` | http://127.0.0.1:3000 | 钉钉外壳 + 身份三门 |
| **仅前端 Mock** | `npm run dev:mock` | :3000 | 同上 |
| **试点单页 H5** | `npm run pilot:dingtalk` | :8080 | 无外壳，彩色身份三门 |

> 若 3000 被占用，Mock 会自动换端口（看终端里的 `Local:` 地址）。

### 最快预览（无需后端）

```bash
cd ~/Downloads/text/和拍---新人入职社交互助平台
cp .env.mock .env.local
npm run dev:mock
```

浏览器打开终端显示的地址 → 左侧点 **「和 新人入职」**。

## 二、页面结构总览

```
钉钉电脑端外壳（demo 模式）
└── 和拍插件
    ├── 身份选择（我是新人 / 导师 / HR）
    ├── 新人首次：登录 → 大礼包 → 认识团队 → 盲盒(8题) → 安全屋
    └── 功能页（顶栏切换）
        ├── 安全屋（首页）
        ├── 我的工位
        ├── 蹭饭地图
        ├── 带教导师
        ├── 闪光星罐 / 兴趣匹配（子页）
        └── AI HR 助手（右侧悬浮）
```

## 三、主要页面说明

### 1. 钉钉外壳（仅 demo / dev:stack）

- 模拟钉钉 PC：消息、工作台、日历等
- 左侧 **「和 新人入职」** 进入和拍

### 2. 身份选择

- **我是新人** → 入职流程
- **我是导师** → 导师工作台
- **我是 HR** → 管理看板

### 3. 新人首次接入

| 顺序 | 页面 | 内容 |
|------|------|------|
| ① | 登录 | 工号 E00001 / 密码 123456 |
| ② | 入职大礼包 | 弹窗「开心收下」 |
| ③ | 认识团队 | 导师、HR、同批伙伴卡片 |
| ④ | 入职盲盒 | 8 道人格题 → 生成面具 |
| ⑤ | 安全屋 | 新人首页 |

### 4. 新人功能页（盲盒完成后）

| 页面 | 功能 |
|------|------|
| **安全屋** | 面具资料、情绪能量条、午餐/导师/工位入口 |
| **我的工位** | 贴画桌面、餐券/星星收集 |
| **蹭饭地图** | 午餐匹配、饭搭子 |
| **带教导师** | 导师列表与状态 |
| **闪光星罐** | 记录今日高光 |
| **AI HR** | 制度问答 + 融入建议（悬浮按钮） |

### 5. 导师端

- **导师工作台**：分配新人列表、面具标签、风险提示

### 6. HR 端

- **HR 仪表盘**：统计、情绪趋势、新人表、入职登记

### 7. 试点模式（8080 / 云部署）

- **无钉钉外壳**，打开即身份选择或登录
- 用于 Render、Oracle、钉钉 H5 真机
